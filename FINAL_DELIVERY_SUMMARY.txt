✅ `php-payments/razorpay_create_order.php` - Order creation
   - Creates Razorpay order via API
   - Stores order in database
   - Returns order_id for checkout

✅ `php-payments/razorpay_verify.php` - Payment verification
   - HMAC-SHA256 signature verification
   - Updates order status to 'paid'
   - Prevents fraud/spoofing

✅ `php-payments/razorpay_webhook.php` - Event handling
   - Listens for payment events
   - Handles: payment.authorized, payment.failed, refund.created
   - Updates database automatically

### BACKEND - STRIPE (USD/Global Payments)
✅ `php-payments/stripe_create_session.php` - Session creation
   - Creates Stripe Checkout Session
   - Stores order in database
   - Returns session for redirect

✅ `php-payments/stripe_webhook.php` - Event handling
   - Listens for payment completion
   - Handles: checkout.session.completed, charge.refunded, disputes
   - Updates database automatically

## 📊 Database Design

### 4 Tables Included
1. **orders** - Core payment transactions
   - 13 columns for complete payment info
   - 5 indexes for performance
   - Tracks: provider, amount, currency, method, status

2. **refunds** - Refund operations
   - 8 columns
   - Foreign key to orders
   - Tracks: refund_id, amount, reason, status

3. **disputes** - Chargebacks & disputes
   - 9 columns
   - Evidence tracking
   - Status management

4. **webhook_logs** - Audit trail
   - 8 columns
   - All webhook events logged
   - Signature verification tracking
